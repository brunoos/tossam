TOSSAM - TinyOS Serial AM for Lua

TOSSAM is a library to send and receive TinyOS's ActiveMessage using the serial port. For example:

local tossam = require("tossam") 

local micaz = tossam.connect("/dev/ttyUSB1", "micaz") 

micaz:register [[ 
  nx_struct msg_serial [50] { 
    nx_uint8_t  sum; 
    nx_uint8_t  count; 
    nx_uint16_t vet[3]; 
  }; 
]]

for i = 1, 3 do 
  local msg, err = micaz:receive() 
  print(msg.sum) 
  print(msg.count) 
  for k, v in ipairs(msg.vet) do
    print(k, v)
  end 
  print("---") 
  msg = { sum = 0, count = i, vet = {10,5,10} }
  micaz:send(msg, 50)
end

micaz:close()

Install

TOSSAM uses Lua 5.1 and the packages:

  • bitlib 23
  • lpeg 0.12
  • struct 1.4
  • luars232 1.0.3

These packages are available in Luarocks.

Message Format

The message's structure is loaded dynamically and it follows the definition of a nx_struct from TinyOS with a minor change. Every AM message has an numeric identifier, you have to specify this identifies after the name of the struture (between square braces) in order to associate the serial message with the nx_struct definition.

TOSSAM supports the nx types from TinyOS:

  • Unsigned integer, big-endian: nx_uint8_t, nx_uint16_t, nx_uint32_t, nx_uint64_t.
  • Unsigned integer, little-endian: nxle_uint8_t, nxle_uint16_t, nxle_uint32_t, nxle_uint64_t.
  • Signed integer, big-endian: nx_int8_t, nx_int16_t, nx_int32_t, nx_int64_t.
  • Signed integer, little-endian: nxle_int8_t, nxle_int16_t, nxle_int32_t, nxle_int64_t.
  • Single precision float-point: nx_float.

Every message is received as a table with the fields defined in the nx_struct. Besides, TOSSAM put in the index 1 the identifier and in the index 2 the name of the nx_struct.

Arrays are also mapped to tables, however, this version supports only unidimensional arrays.

You can specify arrays with fixed or variable sizes. In the last case, the size should be a variable already defined in the struture, for example:

nx_struct my_arrays [20] {
  nx_uint8_t size;
  nx_uint8_t fix[3];
  nx_uint8_t variable[size];
};

API

tossam.connect(port, baud)

Connect to a mote in the serial port and return an object that represents this connection.

port is a string with serial port identification. baud can be a number for the serial baud rate or a string indicating the mote:

  • eyesifx
  • intelmote2
  • iris
  • mica
  • mica2
  • mica2dot
  • micaz
  • shimmer
  • telos
  • telosb
  • tinynode
  • tmote
  • ucmini
obj:close()

Closes the serial connection.

obj:register(def)

Register nx_structs on the object. It returns true for success, or false, followed by an error message, otherwise.

str is a string that contains one or more nx_structs definitions. You do not need to register all strutures at once, the function can be called several times.

obj:registered()

Returns a table with all the registered strutures. The key is the name of the structure and the value its identifies.

obj:unregister(id)

Unregister a struture. The parameter id can be a number or the name of the structure.

The function returns true for success, or false if the struture was not registered.

obj:send(msg [, id])

Sends a message throght the serial port. msg is a table that contains the pairs (name,value) for each field defined in the struture. This operation is blocking.

The parameter id is the identifier for the message. This argument is optional, you can inform the identifier in the index 1 of the message (number or name of the struture). However, if id passed, the index is not used.

In case of error, the function returns nil followed by the error message.

obj:receive()

Returns the AM message received, or nil followed by an error message. This operation is blocking.
